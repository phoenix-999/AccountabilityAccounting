!!!РЕШЕНИЕ РАЗРАБАТЫВАЛОСЬ ДЛЯ УЧЕБНЫХ ЦЕЛЕЙ (ПРАКТИЧЕСКОЕ ПРИМЕНЕНИЕ ИЗУЧАЕМЫХ ТЕХНОЛОГИЙ).  СОДЕРЖИТ ОШИБКИ И НЕДОРАБОТКИ, А ТАК ЖЕ ПЛОХОЙ ЗАПУТАННЫЙ КОД.

Решение построено с использованием технологий .NET, ADO.NET, WCF, Windows Forms.

Решение содержит 2 службы WCF: AuthenticationService и DataAccessService.
Первая отвечает за аутентификацию клиента, вторая - посредник в работе с базой данных.
Обе службы используют транзакции для обновления данных в базе. 
Так же, каждая служба имеет конфигурационный файл, в котором декларативно описана конфигурация самой службы.
Хостинг служб возможен в любым способом, который поддерживает WCF. При хостинге, скорее всего, необходимо будет редактировать конфигурационный файл хоста с учетом конфигурациооных файлов служб. Так было сделано намеренно. 

DataAccessService.dll содержит модель работы с базой данных. При этом, весь SQL прописан в хранимых процедурах на сервере базы данных.
SQL для формирования базы данных, ее таблиц и хранимых процедур хранится в файле DataAccessService.txt рядом с выше указанной .dll.
В скрипте запроса на создания БД имеется внесения тестового пользователя с именем yurii и паролем "1". В базе данных пароли находятся в зашифрованном состоянии.
На данный момент, строка соединения с БД хранится в классе DBAccess.cs в виде статического поля. При необходимости, строку соединения можно вынести в конфигурациооный и реализовать досутп в виде свойства.

Проект DataAccessService содержит клиентскую часть решения.

На даннй момент в решении не реализована процедура добавления нового пользователя.
Так же, процедуры добавления, изменения и удаления проектов, статей и подотчетников следует предоставлять только пользователиям-админам.
